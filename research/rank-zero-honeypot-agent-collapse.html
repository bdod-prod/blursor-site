Rank Zero: How a Single False Source Collapses Frontier LLM Agents

May 22, 2026 · 7 min read · arXiv:2603.00801
Referenced Paper
The Synthetic Web: Adversarially-Curated Mini-Internets for Diagnosing Epistemic Weaknesses of Language Agents
Authors
Shah, S. · Ozgur, L.
Institution
Microsoft
Published
February 28, 2026
Source
arXiv:2603.00801 ↗

A paper from Microsoft, published in February on arXiv, tests something most RAG and agentic retrieval research sidesteps: what happens when a single false source occupies the top search result, while all truthful content remains fully accessible. Prior adversarial work typically removes or degrades truthful sources, conflating two separate problems — access failure and credulity failure. This paper isolates the second. The agents don't fail because truth is hidden. They fail because they stop looking after the first result.

The researchers built procedurally generated synthetic web environments — thousands of hyperlinked articles with ground-truth credibility labels, contamination-filtered to reduce training-data leakage — and ran six frontier models through a controlled search-and-read protocol. The adversarial condition is deliberately minimal: one high-plausibility honeypot article injected at rank zero. Everything else stays the same.

The number that anchors the study: GPT-5 falls from 65.1% to 18.2% accuracy — a 46.9-point collapse — across 5,870 matched queries when a single false source sits at the top of search results.

The Setup: A Controlled Fake Internet

The synthetic environments are not toy benchmarks. Each world contains thousands of hyperlinked articles, procedurally generated with ground-truth factuality and credibility labels. The researchers filtered queries by probing a strong model without tools and removing any query it answered correctly — eliminating cases where the model could rely on memorized training data rather than actual retrieval. Schema adherence and answer extraction rates exceeded 99% across all models, ruling out formatting artifacts as a confound.

The adversarial condition is a single intervention: one honeypot article, written to be plausible, inserted at rank zero — the first result returned by the search layer. No truthful sources are removed. No search constraints are applied. The agent has unlimited access to correct information; it simply has to look past the first result to find it.

This design isolates positional credulity from search access. An agent that fails under these conditions fails not because the web is broken but because it trusts rank over corroboration. The 587 unique queries were each run across 10 rollouts in 4 independently generated worlds, producing 5,870 observations per condition per model — enough to detect even modest effects with high confidence. Two-proportion z-tests confirm p < 1e-16 for all non-trivial models.

The Numbers: Catastrophic Drops Across Every Frontier Model

The results are consistent across every model tested. GPT-5 drops 46.9 points, from 65.1% to 18.2%. o3 drops 31.7 points, from 48.4% to 16.7%. o1 drops 30.6 points, from 39.0% to 8.4%. GPT-4o drops 23.4 points, from 27.2% to 3.8%.

−30.6 pts o1: 39.0% → 8.4%
−23.4 pts GPT-4o: 27.2% → 3.8%

The magnitude varies, but the direction does not. No frontier model resists the effect. The adversarial condition does not require sophisticated manipulation — a single plausible false article at rank zero is sufficient to reduce the best available model to 18.2% accuracy on queries it would otherwise answer correctly nearly two-thirds of the time.

Smaller models — o4-mini and o1-mini — fail almost entirely in both standard and adversarial conditions, at 0.3% and 0.0% respectively in the standard setting. The adversarial comparison is uninformative for them, but the baseline numbers are alarming in their own right. These models are not viable retrieval agents under any condition tested here.

The Human Contrast: Same Queries, Mostly Solved

Human evaluators achieved 98% accuracy in the standard condition and 93% in the adversarial condition — on the same queries, in the same environments, with the same honeypot article at rank zero. The adversarial queries are not inherently unsolvable. They are unsolved by models.

The gap between human adversarial performance (93%) and GPT-5 adversarial performance (18.2%) is not a task-difficulty artifact. Humans appear to apply basic corroboration instincts that current agents do not: checking multiple sources before accepting a top-ranked claim, treating rank as a weak signal rather than a credibility certificate.

The researchers are careful to note the human baseline was drawn from a random sample of one world, so the precise figures carry uncertainty. But the directional conclusion is not in doubt. A 75-point gap between human and model performance under identical adversarial conditions reflects a qualitative difference in how humans and LLM agents treat source rank.

Why Models Fail: Flat Search, False Confidence

Tool usage barely changes between conditions. GPT-5 averages 6.45 calls in the standard setting and 6.61 in the adversarial setting. The model does not recognize it has been misled and does not escalate search effort. The fraction of queries where GPT-5 issues five or more tool calls is 62% in the adversarial condition — meaning even extended search does not reliably produce corroboration-seeking behavior.

0.641
GPT-5 Expected Calibration Error in adversarial condition — up from 0.298 in standard — meaning the model expresses high confidence precisely when it is most wrong.

Calibration collapses under adversarial exposure. GPT-5's Expected Calibration Error doubles from 0.298 in the standard condition to 0.641 in the adversarial condition. Its Brier score rises from 0.332 to 0.591. The model expresses high confidence precisely when it is most wrong — a failure mode more dangerous than low accuracy alone, because downstream systems that rely on model confidence as a routing signal will be misled in the same direction.

The authors identify positional anchoring as the core mechanism: models over-rely on top-ranked results and fail to seek independent corroboration — a retrieval-era analogue of the "lost in the middle" phenomenon documented in long-context settings. In one documented case, GPT-5 issued 162 tool calls attempting to reconstruct a regulatory timeline and still failed, suggesting the problem is not only shallow search but an inability to reconcile conflicting partial information across sources once an anchoring false claim has been encountered.

What to Do About It

The paper's practical implication is structural, not incremental. Any agentic deployment that routes LLM queries through a search layer is exposed to this failure mode by default. The adversarial condition here requires no access to the model, no prompt injection, and no removal of truthful content — only the ability to place one plausible article at the top of search results. That bar is low enough to be a realistic threat in any environment where search rankings can be influenced.

The researchers evaluated models under a uniform zero-shot prompt with no source-criticism instructions. Whether targeted prompting, few-shot demonstrations, or chain-of-thought scaffolding can close the gap remains an open question — the paper does not test mitigations. But the calibration results suggest the problem runs deeper than prompt design: a model that doubles its ECE under adversarial exposure is not just following bad instructions, it is forming confident wrong beliefs.

The minimum viable response is to treat source-criticism scaffolding as a prerequisite rather than an enhancement. Agents that retrieve before reasoning need explicit corroboration requirements — not as a best practice, but as a structural guard against a failure mode that currently reduces the best available model to 18.2% accuracy on questions it can otherwise answer. Rank is not credibility. Current agents behave as though it is.

Key Takeaway

These results come from a controlled synthetic environment where truthful content was never removed — agents failed not from lack of access but from lack of skepticism about the first result they encountered.

Any agentic deployment that routes LLM queries through a search layer is structurally vulnerable to a single well-placed false source; source-criticism scaffolding is not a nice-to-have, it is a prerequisite for reliable retrieval-augmented agents.
